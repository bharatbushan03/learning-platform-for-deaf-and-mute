System Design · v1.0

ISL Learning App
for the Deaf & Mute

// Bhavishya Pandit · Personal Learning Project · Real-World Problem

Phase 01 — User Input Layer
📷
Camera Module Easy
WebRTC + getUserMedia API
Captures live video feed from user's webcam directly in the browser.
  • Runs entirely in browser — no install needed
  • Streams frames at 30fps to hand detector
  • Handles camera permissions & fallback errors
🖥️
Frontend UI Easy
React.js + Tailwind CSS
The face of the app. User opens camera, sees live feed, and sees translated text output.
  • Live camera preview canvas
  • Real-time text output panel
  • Start/Stop controls
↓
Phase 02 — Computer Vision Layer
🖐️
Hand Detection Easy
MediaPipe Hands
Detects hands in each video frame and extracts 21 landmark points per hand.
  • Runs in browser via WASM — no GPU needed
  • 21 (x,y,z) coordinates per hand
  • Works at real-time speed on CPU
📐
Landmark Extraction Easy
MediaPipe + NumPy
Converts raw hand coordinates into a normalized feature vector for the classifier.
  • Normalize coordinates (0 to 1 range)
  • Flatten 21×3 = 63 features per frame
  • Handle missing/partial hand detections
🧠
Gesture Classifier Medium
Scikit-learn / TFLite
Takes landmark features and classifies them into one of the known ISL signs.
  • Start: Random Forest or SVM (simple)
  • Upgrade: CNN or LSTM for sequences
  • Output: sign label + confidence score
↓
Phase 03 — NLP & Translation Layer
📝
Word Buffer Easy
Python / JS Array
Collects classified sign labels over time and builds a word sequence.
  • Holds last N detected signs
  • Removes duplicates & noise
  • Triggers sentence builder after pause
🔤
Sentence Builder Medium
Rule-based → LLM
Converts sign word sequence into grammatically correct English sentence.
  • Phase 1: Simple rule-based joining
  • Phase 2: Claude/GPT API to fix grammar
  • Output: "I want water" from [I, WANT, WATER]
🔊
Text-to-Speech Easy
Web Speech API
Converts the final sentence to audio output so hearing people can understand.
  • Browser built-in — free, no API needed
  • Configurable voice & speed
  • Optional feature — add later
↓
Phase 04 — Data Layer
🗄️
Dataset Hard
Kaggle + Self-Collected
The fuel for your entire model. Start small — 10 signs, 100 samples each.
  • Search: "ISL dataset Kaggle"
  • Self-record via webcam using OpenCV
  • Store as landmark CSVs — not raw video
  • Label: sign name + frame index
⚙️
Model Training Medium
Google Colab (Free)
Train your classifier on the landmark dataset. Export as lightweight model file.
  • Free GPU on Colab — enough for this
  • Export as .pkl (sklearn) or .tflite
  • Load model in backend on app start
  • Retrain as you add more signs
↓
Phase 05 — Backend Layer
🚀
API Server Medium
FastAPI (Python)
Receives landmark data from frontend, runs classifier, returns sign label.
  • POST /predict → returns sign label
  • WebSocket for real-time streaming
  • Loads ML model on startup
🔗
LLM Integration Medium
Claude API / OpenAI
Optional layer that takes raw sign words and returns a clean sentence.
  • POST /translate → clean sentence
  • Add only in Phase 2
  • Use Claude Haiku — cheapest option
☁️
Deployment Easy
Render / Railway (Free)
Host your FastAPI backend for free. Frontend on Vercel or Netlify.
  • Render free tier — enough for learning
  • Frontend: Vercel (React) — free
  • Add later — build locally first
Data Flow — End to End
Camera → MediaPipe → 21 Landmarks → Normalize → Classifier → Sign Label → Word Buffer → Sentence Builder → Text Output → TTS (optional)
Full Tech Stack
Layer Technology Why This
Hand Tracking MediaPipe Hands Free, runs in browser, no GPU, battle-tested by Google
ML Classifier Scikit-learn → TFLite Start simple with RF/SVM, upgrade to neural net when ready
Backend FastAPI (Python) Fast, simple, async, perfect for ML serving
Frontend React.js You already know web basics — React is industry standard
NLP / LLM Claude Haiku API Cheapest LLM, good enough for sentence correction
Training Google Colab Free GPU — enough for 10-sign classifier
Hosting Render + Vercel Both have free tiers — zero cost to deploy
Speech Output Web Speech API Built into every browser — completely free
Build Order → Start with Phase 2 (MediaPipe hand tracking) first. Get landmarks printing to console. Then build Phase 3 classifier. Then wrap it in Phase 5 backend. Frontend last. Data collection runs parallel to everything. LLM layer is Phase 2 work — don't touch it until classification works clean.

Kill Switch → If gesture classification accuracy stays below 70% after 3 weeks — your data is the problem, not the model. Collect more samples before changing architecture.
